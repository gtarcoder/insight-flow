个人信息助理系统架构图设计说明：

1. 顶部展示系统名称: "个人信息助理系统架构"

2. 以分层方式从上至下排列七个主要功能层:
   - 数据采集层 (Data Collection Layer)
   - 数据处理层 (Data Processing Layer)
   - 数据存储层 (Data Storage Layer)
   - 检索查询层 (Retrieval Layer)
   - 分析推理层 (Analysis Layer)
   - 可视化展示层 (Visualization Layer)
   - 推送服务层 (Notification Layer)

3. 每一层使用不同的浅色填充，并在层内放置相关组件:
   
   数据采集层组件:
   - 微信公众号爬虫
   - 微博爬虫
   - 小红书爬虫
   - B站爬虫
   - 爬虫调度器

   数据处理层组件:
   - LLM处理引擎
   - 文本摘要生成器
   - 主题分类器
   - 情感分析器
   - 实体识别器

   数据存储层组件:
   - MongoDB (文档存储)
   - Qdrant (向量存储)
   - Neo4j (图数据库)

   检索查询层组件:
   - 语义搜索引擎
   - 关键词搜索
   - 相似内容检索

   分析推理层组件:
   - 关系分析引擎
   - 时序分析器
   - 因果推理器

   可视化展示层组件:
   - 关系图生成器
   - 时序图绘制器
   - 因果图生成器

   推送服务层组件:
   - 推送管理器
   - 价值评估引擎
   - 微信推送
   - 邮件推送
   - App推送

4. 使用箭头连接各层之间的数据流动:
   - 数据采集层 → 数据处理层
   - 数据处理层 → 数据存储层
   - 数据存储层 → 检索查询层
   - 检索查询层 → 分析推理层
   - 分析推理层 → 可视化展示层
   - 分析推理层 → 推送服务层

5. 在右侧添加图例:
   - 数据流向
   - 主要组件
   - 外部服务

6. 在底部添加说明文本:
   "系统采用模块化、分层设计，确保可扩展性和可维护性。通过多种数据存储方式和LLM处理能力，实现信息的智能化处理和个性化推送。"
